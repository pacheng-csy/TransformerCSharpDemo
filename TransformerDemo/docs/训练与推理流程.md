## 训练与推理流程（含时序）

本文件从“流程与时序”的角度，分别梳理数字玩具任务与三国问答任务在**训练与推理阶段的调用顺序**，帮助你快速对照代码理解执行路径。

---

### 一、数字玩具任务：训练流程时序

数字任务的训练命令为：

- `dotnet run -- train`

对应的大致调用链与时序如下：

1. **入口选择模式**
   - `Program.Main(args)` 解析命令行，进入 `Train()`。
2. **生成训练/验证数据**
   - `DataGenerator.GenerateAll()`：
     - 随机生成 `TrainCount=1000` 条训练样本与 `ValidCount=200` 条验证样本；
     - 每条样本是 `DataGenerator.Sample`，包含：
       - `Input`：随机长度的 0~9 序列；
       - `Target`：每个元素加一（模 10）。
3. **构造词表与模型**
   - `var vocab = new Vocabulary()`：
     - 定义 `[PAD]/[SOS]/[EOS]` 以及数字 0~9 对应的 token id。
   - `var model = new TransformerModel(...)`：
     - 小尺寸 Transformer（例如 8 维、1 头、1 层 encoder/decoder）；
     - 便于教学与快速收敛。
4. **按 epoch 训练**
   - 在 `Train()` 中循环：
     - `Training.TrainEpochBackprop(model, trainData, MaxLen, BatchSize, vocab)`：
       1. 通过 `DataGenerator.Batches(...)`：
          - 为每条样本添加 `[SOS]/[EOS]`；
          - 用 `[PAD]` 补齐到固定 `maxLen`；
          - 输出 `(inputIds, targetIds, validLengths)` batch。
       2. 对每个 batch 调用 `Training.TrainStepBackprop(...)`：
          - `model.ZeroGrad()`；
          - `model.Forward(...)` 得到 logits；
          - `MatrixHelper.CrossEntropyLoss(...)` 计算 loss；
          - `MatrixHelper.CrossEntropyLossBackward(...)` 得到对 logits 的梯度；
          - `model.Backward(...)` 将梯度反传到所有参数；
          - 使用简单 SGD：`p -= lr * grad`。
     - 同时使用 `Training.Validate(...)` 在验证集上计算平均 loss。
5. **完整验证与指标输出**
   - 训练完成后，可调用：
     - `Evaluation.EvaluateOnValidation(model, validData, MaxLen, BatchSize, vocab)`：
       - 打印验证集平均 loss；
       - 打印 token 级准确率与序列级准确率。
6. **保存模型**
   - `model.SaveToDirectory("SavedModel")`：
     - 写出 `config.txt`（超参数）与 `weights.bin`（按固定顺序展开的权重矩阵）。

### 二、数字玩具任务：推理流程时序

数字任务的推理命令为：

- `dotnet run -- infer`
- 或使用 `InferenceExample.Run(...)` 直接在代码中调用。

大致调用链如下：

1. `Program.Main(args)` 根据参数进入 `RunInfer(dir)`。
2. `InferenceExample.Run(int[][] input, string modelDir)`：
   1. 检查模型目录是否存在。
   2. 使用 `TransformerModel.LoadFromDirectory(modelDir, out vocabSize, out _, out _)`：
      - 读取 `config.txt` 与 `weights.bin`，构造并加载模型；
   3. 创建 `var vocab = new Vocabulary()`；
   4. 对每一条输入整数序列：
      - 根据规则计算期望输出（每位加一）；
      - 构造 `DataGenerator.Sample`；
      - 调用 `DataGenerator.Preprocess(...)` 生成 `(inputIds, targetIds, validLengths)`；
      - 调用 `model.Forward(...)` 得到 logits；
      - 对每个位置取 argmax 得到预测 token id，再用 `vocab.DecodeToDigit` 解码为数字；
      - 在控制台打印“输入 / 期望 / 预测”三行结果。

### 三、三国问答（CPU）：训练与推理时序

三国问答 CPU 训练命令为：

- `dotnet run -- sanguo`

推理命令为：

- `dotnet run -- ask`（或指定模型目录：`dotnet run -- ask SanguoModel`）

#### 1. 训练流程

1. **入口选择模式**
   - `Program.Main(args)` 判断 `args[0] == "sanguo"` 且未指定 `gpu`，进入 `TrainSanguo()`。
2. **解析数据路径**
   - `GetSanguoDataPaths()`：
     - 在运行目录下优先查找 `data/`，其次 `Data/`；
     - 返回训练集与验证集 JSONL 路径。
3. **加载 JSONL 问答与构建字符词表**
   - `SanguoQaDataLoader.Load(trainPath, validPath)`：
     - 调用 `ReadJsonl` 读取 JSONL 文件：
       - 每行解析 `{"question": "...", "answer": "..."}`；
     - 收集所有问题与答案文本，构建 `CharVocabulary`（字符级词表）；
     - 将每条问答转为 `DataGenerator.Sample`，其中：
       - `Sample.Input`：question 的字符 token id 序列；
       - `Sample.Target`：answer 的字符 token id 序列。
4. **构造模型**
   - 使用适合文本任务的超参数（例如：`dModel=64, numHeads=2, dFf=128, numLayers=2`）构造 `TransformerModel`。
5. **按 epoch 训练**
   - 在 `TrainSanguo()` 中循环：
     - `Training.TrainEpochBackpropTokenized(model, trainData, SanguoMaxLen, SanguoBatchSize, SanguoLr)`：
       - 内部通过 `DataGenerator.BatchesTokenized(...)`：
         - 为已编码好的 `Sample.Input` / `Sample.Target` 添加 `[SOS]/[EOS]`；
         - 用 `[PAD]` 填充到固定长度；
         - 输出 `(inputIds, targetIds, validLengths)`；
       - 对每个 batch 执行与数字任务相同的反向传播 + SGD 更新逻辑。
     - 使用 `Training.ValidateTokenized(...)` 在验证集上计算平均 loss。
6. **保存模型与字符词表**
   - 训练完成后：
     - `TransformerModel.SaveToDirectory(SanguoModelDir)`；
     - `CharVocabulary.SaveToDirectory(SanguoModelDir)`（写出 `vocab.json`）。

#### 2. 推理与时序

推理由 `SanguoInference` 负责，核心方法为：

1. **加载模型与词表**
   - `SanguoInference.LoadFromDirectory(modelDir)`：
     - 调用 `TransformerModel.LoadFromDirectory(...)` 加载模型；
     - 调用 `CharVocabulary.LoadFromDirectory(...)` 加载 `vocab.json`；
     - 返回 `(model, maxLen, vocab)`。
2. **交互式问答入口**
   - `SanguoInference.RunInteractive(modelDir)`：
     - 在控制台循环读取用户输入问题；
     - 对每个问题调用 `Ask(model, vocab, question, maxLen)`；
     - 打印生成的答案文本。
3. **自回归解码时序（Ask 内部）**
   - `Ask(TransformerModel model, CharVocabulary vocab, string question, int maxLen)`：
     1. 使用 `vocab.EncodeText(question)` 将问题编码为字符 id；
     2. 构造 encoder 输入序列：
        - `[SOS] + question_ids + [EOS] + PAD...`，长度固定为 `maxLen`；
     3. 将 encoder 输入封装为 batch（大小为 1）：
        - `encBatch: int[][]`、`encValidLengths: int[]`；
     4. 初始化 decoder 输入：
        - 第一个位置为 `[SOS]`，其余位置为 `[PAD]`；
     5. 从 `step = 0` 开始循环，最多执行 `maxLen-1` 步：
        - 构造 `decBatch` 与 `decValidLengths`；
        - 调用 `model.Forward(encBatch, decBatch, encValidLengths, decValidLengths)`；
        - 取当前 step 的 logits，使用 argmax 选出下一个 token id；
        - 如为 `[EOS]` 则停止，否则：
          - 将该 id 追加到输出序列；
          - 写回到 `decIds[step+1]`，继续下一步。
     6. 将生成的 token id 序列用 `vocab.DecodeIds(...)` 解码为中文答案。

### 四、三国问答 GPU 训练时序（可选）

当你希望在 GPU 上训练更大模型时，可以使用：

- 编译：`dotnet build -p:UseGpu=true`
- 运行：`dotnet run -- sanguo gpu`

对应的大致时序为：

1. `Program.Main(args)` 检测到 `args[0] == "sanguo" && args[1] == "gpu"`，进入 `TrainSanguoGpu()`。
2. `GetSanguoDataPaths()` 解析训练/验证 JSONL 路径。
3. `SanguoQaDataLoader.Load(...)` 加载问答数据与构建字符词表。
4. 调用 `SanguoGpuTrainer.TrainOnGpu(...)`：
   - 使用 TorchSharp 的 `Seq2SeqTransformer` 在 CUDA 设备上训练；
   - 训练循环中仍然使用 `DataGenerator.BatchesTokenized(...)` 生成 batch。
5. 训练完成后，`SanguoGpuTrainer.ExportToCSharpFormat(...)`：
   - 将 TorchSharp 模型权重提取出来；
   - 重新排布为与 C# `TransformerModel.GetAllParameters()` 顺序兼容的二维数组；
   - 调用 `TransformerModel.SaveToDirectory(outputDir)` 与 `CharVocabulary.SaveToDirectory(outputDir)`。

之后可以像 CPU 训练一样，使用 `SanguoInference.RunInteractive(outputDir)` 进行问答推理。
