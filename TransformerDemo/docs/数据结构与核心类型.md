## 数据结构与核心类型说明

本文件整理项目中最重要的**数据结构（record / class）与核心方法**，帮助你在阅读代码时快速对照理解。

---

### 一、数据相关类型（Data 层）

#### 1. `DataGenerator.Sample`

```text
public record Sample(int[] Input, int[] Target);
```

- 抽象了一条“序列到序列”的样本：
  - `Input`：源序列（数字任务中是整数 0~9；三国问答中是已编码的字符 id）；
  - `Target`：目标序列（“每位加一”的结果，或回答文本的 token id）。
- 在数字任务中，`Input`/`Target` 是**原始数字**；
- 在三国问答任务中，`Input`/`Target` 是**已通过 `CharVocabulary` 编码的 token id**。

#### 2. `Vocabulary`（数字任务词表）

关键字段：

- `PadId = 0`、`SosId = 1`、`EosId = 2`；
- 数字 0~9 对应 token 3~12。

关键方法：

- `int EncodeDigit(int digit)`：0~9 → token id；
- `int DecodeToDigit(int tokenId)`：token id → 数字（非数字 token 返回 -1）；
- `bool IsPad(int tokenId)`、`bool IsSpecial(int tokenId)`：判断是否为特殊 token。

#### 3. `CharVocabulary`（三国问答字符级词表）

关键字段：

- `PadId = 0`、`SosId = 1`、`EosId = 2`；
- 从 `FirstCharTokenId = 3` 开始为所有出现过的字符分配 id。

关键方法：

- `void BuildFromTexts(IEnumerable<string> texts)`：
  - 扫描所有文本，收集字符集合并按顺序生成 id；
- `int[] EncodeText(string text)`：
  - 将整段文本编码为 token id 数组（仅内容，不含 `[SOS]/[EOS]`）；
- `string DecodeIds(int[] ids)`：
  - 将 token id 数组解码为字符串，自动跳过 `[PAD]/[SOS]/[EOS]`；
- `void SaveToDirectory(string path)` / `void LoadFromDirectory(string path)`：
  - 将字符列表保存到 `vocab.json`，或从中恢复，确保训练与推理共用同一词表。

#### 4. `SanguoQaDataLoader.QaRecord`

```text
public record QaRecord(string Question, string Answer);
```

- 对应 JSONL 中每一行的结构：
  - `{"question": "...", "answer": "..."}`；
- 用于中间表示，随后被编码为 `Sample`。

关键方法：

- `List<QaRecord> ReadJsonl(string path, int? take = null)`：
  - 从 JSONL 文件逐行读取问答记录；
- `(List<Sample> train, List<Sample> valid, CharVocabulary vocab) Load(string trainPath, string validPath)`：
  - 读取训练/验证 JSONL；
  - 构建 `CharVocabulary`；
  - 将问答文本编码为 `DataGenerator.Sample` 列表返回。

### 二、模型相关类型（Model 层）

#### 1. `TransformerModel`

构造函数：

```text
public TransformerModel(
    int vocabSize,
    int dModel,
    int numHeads,
    int dFf,
    int numEncoderLayers,
    int numDecoderLayers,
    int maxLen)
```

内部组合：

- `_embedding`：共享的 token Embedding；
- `_encoder`：多层 Encoder（含位置编码、多头注意力、前馈网络等）；
- `_decoder`：多层 Decoder（带因果 Mask 与 Cross-Attention）；
- `_outputProj`：线性层，将隐藏状态映射到词表大小的 logits。

关键方法：

- `float[][][] Forward(int[][] encIds, int[][] decIds, int[] encValidLengths, int[] decValidLengths)`：
  - 输入：
    - `encIds` / `decIds`：形状为 `(batch, seqLen)` 的 token id；
    - `encValidLengths` / `decValidLengths`：每条序列的有效长度（不含 `[PAD]`）；
  - 输出：
    - `logits`：形状为 `(batch, decLen, vocabSize)` 的 3D 数组。
- `void ZeroGrad()`：
  - 清空所有可训练参数上的梯度，为下一步反向传播做准备。
- `void Backward(float[][][] dLdLogits, int[][] encIds, int[][] decIds)`：
  - 手写反向传播入口，将对 logits 的梯度一路传回到 Embedding 与各层权重。
- `List<float[][]> GetAllParameters()` / `List<float[][]> GetAllGradients()`：
  - 以固定顺序收集所有参数或梯度矩阵，用于：
    - 数值梯度优化（SPSA）；
    - SGD 手动更新；
    - 权重保存/加载。
- `float[] GetParametersFlat()` / `void SetParametersFromFlat(float[] flat)`：
  - 将所有参数展平为一维数组或从一维数组恢复；
  - 便于实现“参数向量”层面的优化算法。
- `void SaveToDirectory(string path)` / `void LoadFromDirectory(string path)`：
  - 将模型结构与权重保存到目录，或从中恢复；
  - 存储格式：
    - `config.txt`：超参数与结构信息；
    - `weights.bin`：按矩阵顺序序列化的 float32 权重。
- `static (TransformerModel model, int maxLen) LoadFromDirectory(...)`：
  - 工厂方法：从目录直接构造出完整模型实例。

### 三、训练与评估核心方法（Training 层）

#### 1. 损失与单步训练

- `float ComputeLoss(TransformerModel model, int[][] encIds, int[][] decIds, int[] encValidLengths, int[] decValidLengths, int[][] targetIds)`：
  - 前向计算 logits；
  - 使用 `MatrixHelper.CrossEntropyLoss` 计算忽略 `[PAD]` 位置的平均交叉熵 loss。

- `float TrainStepBackprop(...)`：
  - 完整执行一次“前向 → 计算 loss → 反向 → SGD 更新”的训练步骤；
  - 是所有基于反向传播训练的基础单元。

#### 2. 按 epoch 训练（数字任务 / 文本任务）

- `float TrainEpochBackprop(TransformerModel model, IReadOnlyList<Sample> trainData, int maxLen, int batchSize, Vocabulary vocab, float lr)`：
  - 面向数字任务；
  - 内部通过 `DataGenerator.Batches(...)` 生成 batch 并多次调用 `TrainStepBackprop`。

- `float TrainEpochBackpropTokenized(TransformerModel model, IReadOnlyList<Sample> trainData, int maxLen, int batchSize, float lr)`：
  - 面向三国问答等文本任务；
  - 使用 `DataGenerator.BatchesTokenized(...)` 处理已编码好的 `Sample.Input` / `Sample.Target`。

#### 3. 验证与指标

- `float Validate(...)` / `float ValidateTokenized(...)`：
  - 对验证集做前向计算并返回平均 loss；
  - 前者用于数字任务，后者用于文本任务。

- `float TokenAccuracy(...)`：
  - 计算 Token 级准确率：所有非 `[PAD]` 位置上预测类别与目标类别完全相同的比例。

- `void Evaluation.EvaluateOnValidation(...)`：
  - 组合调用验证与准确率计算；
  - 在控制台输出：
    - 验证集平均 loss；
    - Token 级准确率（平均）；
    - 序列级准确率（平均）。

### 四、推理与 GPU 训练相关方法

#### 1. 三国问答推理：`SanguoInference`

关键方法：

- `(TransformerModel model, int maxLen, CharVocabulary vocab) LoadFromDirectory(string path)`：
  - 从目录加载训练好的模型与字符词表；
  - 要求路径下存在 `config.txt`、`weights.bin` 与 `vocab.json`。
- `string Ask(TransformerModel model, CharVocabulary vocab, string question, int maxLen)`：
  - 给定一个中文问题字符串：
    - 编码为字符 token id；
    - 使用自回归方式逐步生成答案 token；
    - 解码为中文答案文本返回。
- `void RunInteractive(string modelDir)`：
  - 控制台交互式问答循环，方便快速体验模型效果。

#### 2. 三国问答 GPU 训练：`SanguoGpuTrainer`

关键入口：

- `void TrainOnGpu(IReadOnlyList<Sample> trainData, IReadOnlyList<Sample> validData, CharVocabulary vocab, int maxLen, int batchSize, int epochs, double learningRate, int dModel, int numHeads, int dFf, int numEncoderLayers, int numDecoderLayers, string outputDir)`：
  - 使用 TorchSharp 构造与 C# `TransformerModel` 结构对应的 `Seq2SeqTransformer`；
  - 在 CUDA 上迭代训练，并在验证集上评估；
  - 训练结束后将权重导出为 C# 模型可用的格式写入 `outputDir`。

内部辅助：

- `ExportToCSharpFormat(...)`：
  - 读取 TorchSharp 模型各层权重；
  - 按 `TransformerModel.GetAllParameters()` 的顺序与布局写入 C# 模型；
  - 最终调用 `csharpModel.SaveToDirectory(outputDir)` 与 `vocab.SaveToDirectory(outputDir)`。
