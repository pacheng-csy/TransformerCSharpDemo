## 架构总览

本项目是一个**纯 C# 实现的小型 Transformer 教学 Demo**，包含两个主要任务：

- **数字玩具任务**：输入一串 0~9，输出“每位加一”的整数序列；
- **三国演义问答任务**：基于 JSONL 问答数据，训练字符级 Seq2Seq 模型，支持 CPU 与可选 GPU 训练。

整体架构遵循经典的 “数据 → 模型 → 训练 → 评估 → 推理” 流水线，并在此基础上扩展出三国问答分支。

---

### 1. 模块分层概览

- **入口层（Program）**
  - 解析命令行参数，选择运行模式：
    - 数字任务：`train` / `infer`；
    - 三国问答：`sanguo`（CPU 训练）、`sanguo gpu`（GPU 训练）、`ask`（交互式问答）；
    - 数据生成：`generate-sanguo`。

- **数据层（Data + data 目录）**
  - 负责：
    - 数据的**生成或加载**；
    - 词表构建与 token 编码；
    - 添加 `[SOS]/[EOS]/[PAD]` 并按 batch 输出给模型。
  - 关键组件：
    - `Vocabulary`、`DataGenerator`：服务于数字任务；
    - `CharVocabulary`、`SanguoQaDataLoader`、`SanguoQaDataGenerator`：服务于三国问答任务；
    - `data/sanguo_qa_*.jsonl`：问答数据文件。

- **核心数学与算子层（Core）**
  - `MatrixHelper`：实现矩阵乘法、Softmax、LayerNorm、交叉熵及其反向传播，是手写 Transformer 的“算子库”。

- **模型层（Model）**
  - 纯 C# 实现的 Transformer 结构：
    - Embedding、位置编码、多头注意力、前馈网络、Encoder/Decoder 层；
    - `TransformerModel` 封装前向、反向、参数管理与权重持久化。

- **训练与评估层（Training）**
  - 负责将数据与模型组织成训练循环：
    - 基于数值梯度的 SPSA；
    - 手写反向传播 + SGD；
    - 验证集 Loss、Token 级/序列级准确率评估。
  - 对文本任务提供额外的 `TrainEpochBackpropTokenized` / `ValidateTokenized` 接口。

- **推理层（Inference）**
  - 数字任务：`InferenceExample`，演示如何加载保存的模型并做推理；
  - 三国问答：`SanguoInference`，封装加载模型/词表与 `Ask` 问答接口。

- **GPU 扩展层（SanguoGpuTrainer + TorchSharp）**
  - 当使用 `-p:UseGpu=true` 编译时，引入 TorchSharp + CUDA：
    - 使用 PyTorch 风格 API 在 GPU 上训练更大的模型；
    - 训练结束后按 C# 模型的参数顺序导出权重，供纯 C# 推理使用。

---

### 2. 数据流与控制流总览

从命令行到模型推理的高层数据流可以简化为：

```
命令行参数 → Program.Main
  ├─ 数字任务:
  │   ├─ DataGenerator.GenerateAll()                // 生成 (train, valid) 数字样本
  │   ├─ DataGenerator.Batches(...)                 // 加 [SOS]/[EOS]/[PAD]，按批次输出
  │   ├─ Training.TrainEpochBackprop(...)          // 反向传播 + SGD 训练
  │   ├─ Evaluation.EvaluateOnValidation(...)      // 在验证集评估
  │   ├─ TransformerModel.SaveToDirectory(...)     // 保存权重与配置
  │   └─ InferenceExample.Run(...)                 // 载入模型，对整数序列推理
  │
  └─ 三国问答任务:
      ├─ SanguoQaDataLoader.Load(...)              // 读取 JSONL，构建 CharVocabulary 与 Sample 列表
      ├─ DataGenerator.BatchesTokenized(...)       // 基于已编码 token，添加 [SOS]/[EOS]/[PAD]
      ├─ Training.TrainEpochBackpropTokenized(...) // CPU 训练
      ├─ Training.ValidateTokenized(...)           // CPU 验证
      ├─ 或 SanguoGpuTrainer.TrainOnGpu(...)       // TorchSharp GPU 训练并导出权重
      ├─ TransformerModel.SaveToDirectory(...)     // 保存模型权重
      ├─ CharVocabulary.SaveToDirectory(...)       // 保存字符词表 vocab.json
      └─ SanguoInference.Ask(...) / RunInteractive // 加载模型与词表，完成问答推理
```

从图中可以看到：

- **数据层**负责把“人类可读”的数字序列 / 中文文本，变成带特殊 token 的 ID 序列；
- **模型层**只关心 `int[][]` 形式的 encoder/decoder 输入与 valid length，不关心是“数字”还是“中文字符”；
- **训练层**负责调用模型完成前向/反向、计算损失并更新参数；
- **推理层**在训练完成后，加载保存的模型与词表，对新输入做解码。

---

### 3. 两条任务分支在同一架构下的复用点

- **统一的数据表示**：无论是数字任务还是三国问答，最终都会转换成 `DataGenerator.Sample` 与 `int[][]` 形式的 batch；
- **统一的模型结构**：两条任务都使用同一份 `TransformerModel` 实现，只是超参数与词表不同；
- **统一的训练框架**：`Training` 中的核心训练循环与验证逻辑对任务类型无感；
- **差异集中在数据与词表**：
  - 数字任务使用 `Vocabulary` + 内存随机生成；
  - 三国问答使用 `CharVocabulary` + JSONL 文件加载。

理解这一点，有助于你在现有架构上继续增加新的任务（例如其他文本数据集），而**无需重写模型与训练框架**。
