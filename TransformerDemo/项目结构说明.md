# TransformerDemo 项目结构说明

本说明帮助快速定位代码、按“数据 → 模型 → 训练/评估”的顺序阅读学习。

---

## 目录与文件分类

```
TransformerDemo/
├── Program.cs                 # 程序入口：推理 / 训练（反向传播 或 SPSA）
├── 项目结构说明.md            # 本文件
│
├── Core/                      # 基础数学与反向传播工具
│   └── MatrixHelper.cs        # 矩阵运算、Softmax、LayerNorm、交叉熵及对应 Backward
│
├── Data/                      # 词表与数据
│   ├── Vocabulary.cs          # 词表：PAD/SOS/EOS + 数字 0~9 的 token id 映射
│   └── DataGenerator.cs       # 随机生成“每位加一”样本、预处理、按 batch 迭代
│
├── Model/                     # Transformer 模型结构（建议按依赖顺序阅读）
│   ├── Embedding.cs           # 词嵌入层（查表 + Backward 散射）
│   ├── PositionalEncoding.cs  # 正弦/余弦位置编码（无参数，梯度直通）
│   ├── ScaledDotProductAttention.cs  # 缩放点积注意力（单头）+ Backward
│   ├── MultiHeadAttention.cs  # 多头注意力（线性投影 + 多 SDPA + 输出投影 + Backward）
│   ├── LayerNorm.cs           # 层归一化（3D）+ Backward
│   ├── PositionwiseFeedForward.cs    # 两层线性 + ReLU + Backward
│   ├── EncoderLayer.cs        # 编码器层：Self-Attn + Add&Norm + FFN + Add&Norm + Backward
│   ├── DecoderLayer.cs        # 解码器层：Masked Self-Attn + Cross-Attn + FFN（各带 Add&Norm）+ Backward
│   ├── Encoder.cs             # 编码器：PE + 堆叠 EncoderLayer + Backward
│   ├── Decoder.cs             # 解码器：PE + 堆叠 DecoderLayer + Backward
│   └── TransformerModel.cs    # 整体：Embedding → Encoder/Decoder → 输出投影；ZeroGrad/Backward/保存加载
│
├── Training/                  # 训练与评估
│   ├── Training.cs            # 损失计算、SPSA 单步、反向传播+SGD 单步/按 epoch 训练、验证、Token 准确率
│   └── Evaluation.cs          # 验证集上的完整评估（loss、Token/序列准确率）
│
└── Inference/                 # 推理
    └── InferenceExample.cs    # 加载已保存模型并做推理示例
```

---

## 建议阅读顺序（学习用）

1. **Program.cs** — 弄清入口与三种模式（infer / 默认训练 / spsa）。
2. **Data/** — Vocabulary、DataGenerator，理解“数字序列 → token id → batch”的流程。
3. **Core/MatrixHelper.cs** — 前向运算与 Backward 的写法（矩阵乘、Softmax、LayerNorm、CE）。
4. **Model/** — 按上面列表自上而下：Embedding → 注意力与 FFN → Encoder/Decoder 层 → TransformerModel。
5. **Training/** — Training（loss、SPSA、Backprop+SGD）、Evaluation（指标计算）。
6. **Inference/** — 如何加载权重并跑推理。

## 命名空间说明

所有代码均使用命名空间 `TransformerDemo`，未按子目录拆分子命名空间，因此移动文件后**无需**修改 `using`，编译即可通过。
