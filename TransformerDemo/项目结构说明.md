## TransformerDemo 项目结构说明

本说明帮助你从整体上理解项目结构，并按“数据 → 模型 → 训练/评估 → 推理”的顺序学习。  
项目包含两个主要任务：

- **数字玩具任务**：输入一串 0~9，输出“每位加一”的整数序列；
- **三国演义问答任务**：基于 JSONL 数据训练一个字符级问答模型（支持 CPU 与可选 GPU 训练）。

---

### 一、目录与文件分类（总览）

下面的结构以 `TransformerDemo` 项目目录为根：

```
TransformerDemo/
├── Program.cs                 # 程序入口：选择运行模式（数字任务 / 三国训练 / 生成数据 / 问答推理）
├── TransformerDemo.csproj     # 项目配置（net9.0、可选 TorchSharp GPU 支持）
├── 项目结构说明.md            # 本文件（项目结构总览）
│
├── data/                      # 三国问答数据文件（JSONL，运行时复制或由生成器产出）
│   ├── sanguo_qa_train.jsonl  # 问答训练集
│   ├── sanguo_qa_valid.jsonl  # 问答验证集
│   └── README.md              # 三国问答数据格式说明
│
├── Core/                      # 基础数学与反向传播工具
│   └── MatrixHelper.cs        # 矩阵运算、Softmax、LayerNorm、交叉熵及对应 Backward
│
├── Data/                      # 词表与数据加载/预处理
│   ├── Vocabulary.cs          # 数字任务词表：PAD/SOS/EOS + 数字 0~9 的 token id 映射
│   ├── DataGenerator.cs       # 数字任务：随机生成“每位加一”样本、预处理、按 batch 迭代
│   ├── CharVocabulary.cs      # 三国问答：字符级词表（PAD/SOS/EOS + 所有出现过的字符）
│   ├── SanguoQaDataLoader.cs  # 从 JSONL 读取三国问答数据，构建字符词表并转为 Sample 列表
│   └── SanguoQaDataGenerator.cs # 生成三国问答 JSONL 数据（固定模板 + 扩展问答，写出 train/valid）
│
├── Model/                     # Transformer 模型结构（建议按依赖顺序阅读）
│   ├── Embedding.cs           # 词嵌入层（查表 + Backward 散射）
│   ├── PositionalEncoding.cs  # 正弦/余弦位置编码（无参数，梯度直通）
│   ├── ScaledDotProductAttention.cs  # 缩放点积注意力（单头）+ Backward
│   ├── MultiHeadAttention.cs  # 多头注意力（线性投影 + 多 SDPA + 输出投影 + Backward）
│   ├── LayerNorm.cs           # 层归一化（3D）+ Backward
│   ├── PositionwiseFeedForward.cs    # 两层线性 + ReLU + Backward
│   ├── EncoderLayer.cs        # 编码器层：Self-Attn + Add&Norm + FFN + Add&Norm + Backward
│   ├── DecoderLayer.cs        # 解码器层：Masked Self-Attn + Cross-Attn + FFN（各带 Add&Norm）+ Backward
│   ├── Encoder.cs             # 编码器：位置编码 + 堆叠 EncoderLayer + Backward
│   ├── Decoder.cs             # 解码器：位置编码 + 堆叠 DecoderLayer + Backward
│   └── TransformerModel.cs    # 整体：Embedding → Encoder/Decoder → 输出投影；ZeroGrad/Backward/保存/加载
│
├── Training/                  # 训练与评估
│   ├── Training.cs            # 损失计算、SPSA、反向传播+SGD 单步/按 epoch 训练、验证、Token 准确率
│   ├── Evaluation.cs          # 在完整验证集上的评估（loss、Token/序列准确率）
│   └── SanguoGpuTrainer.cs    # 三国问答 GPU 训练（TorchSharp + CUDA，导出权重给 C# 推理使用）
│
├── Inference/                 # 推理
│   ├── InferenceExample.cs    # 数字任务：加载已保存模型，对若干整数序列做“每位加一”推理示例
│   └── SanguoInference.cs     # 三国问答：加载模型与字符词表，提供 Ask(question) 与交互式问答
│
└── docs/                      # 教学文档（架构 / 流程 / 时序 / 数据结构 等，便于他人学习）
```

### 二、核心模块与任务说明

- **Program.cs（程序入口）**
  - 负责解析命令行并选择运行模式：
    - `train`：数字任务训练（反向传播 + SGD 或可选 SPSA）；
    - `infer [目录]`：数字任务推理，加载指定目录下的模型权重；
    - `sanguo`：三国问答 CPU 训练；
    - `sanguo gpu`：三国问答 GPU 训练（需要 `-p:UseGpu=true` 编译，并安装 TorchSharp + CUDA）；
    - `generate-sanguo`：调用 `SanguoQaDataGenerator` 生成 JSONL 训练/验证数据；
    - `ask [模型目录]`：运行 `SanguoInference.RunInteractive`，从控制台问答。

- **Data 层（Vocabulary / CharVocabulary / DataGenerator / SanguoQaData*）**
  - 数字任务：
    - `Vocabulary`：为 0~9 分配稳定的 token id，并定义 `[PAD]/[SOS]/[EOS]`；
    - `DataGenerator`：
      - `GenerateAll` 在内存中构造 1000 条训练样本、200 条验证样本；
      - `Preprocess` / `Batches` 负责加上 `[SOS]/[EOS]`、填充 `[PAD]`，并生成 batch。
  - 三国问答任务：
    - `CharVocabulary`：从所有问答文本构建字符级词表，并支持保存/加载 `vocab.json`；
    - `SanguoQaDataLoader`：
      - 读取 `sanguo_qa_train.jsonl` / `sanguo_qa_valid.jsonl`；
      - 解析 `question` / `answer`，用 `CharVocabulary` 编码为 token id；
      - 组装为 `DataGenerator.Sample` 列表，供训练与验证使用；
    - `SanguoQaDataGenerator`：按预设模板构造 1200 条左右的问答，对应训练/验证集写入 JSONL。

- **Model 层（纯 C# Transformer 实现）**
  - 由 `Embedding`、`Encoder`、`Decoder`、多头注意力、前馈网络、LayerNorm 等模块组成；
  - `TransformerModel` 封装前向 `Forward`、反向 `Backward`、参数打平与还原、权重保存/加载等。

- **Training / Evaluation（训练与评估）**
  - `Training`：
    - 提供基于数值梯度的 **SPSA** 与手写反向传播的 **SGD** 两条训练路径；
    - 既支持数字任务使用的 `Batches`，也支持三国问答使用的 `BatchesTokenized`。
  - `Evaluation`：
    - 在完整验证集上计算平均 loss；
    - 统计 **Token 级准确率** 与 **序列级准确率**，帮助直观评估模型好坏。

- **Inference（推理）**
  - 数字任务：`InferenceExample.Run` 加载保存的模型，对给定若干整数序列做“每位加一”预测。
  - 三国问答：`SanguoInference` 提供：
    - `LoadFromDirectory`：从目录加载模型和字符词表；
    - `Ask`：自回归解码，一个 token 一个 token 生成答案；
    - `RunInteractive`：命令行交互式问答工具。

### 三、两类任务对比：数字玩具 vs 三国问答

- **数据来源**
  - 数字任务：运行时在内存中随机生成，不落地文件；
  - 三国问答：通过 JSONL 文件存储，便于版本管理与人工编辑。

- **词表与编码方式**
  - 数字任务：`Vocabulary` 将 0~9 编码为 token 3~12；
  - 三国问答：`CharVocabulary` 为所有出现过的字符分配 id，完全字符级建模。

- **预处理与 batch 形成**
  - 数字任务：`DataGenerator.Preprocess` / `Batches`；
  - 三国问答：`SanguoQaDataLoader.Load` → `DataGenerator.PreprocessTokenized` / `BatchesTokenized`。

- **训练入口**
  - 数字任务：`Program.Train` → `Training.TrainEpochBackprop` + `Evaluation.EvaluateOnValidation`；
  - 三国 CPU：`Program.TrainSanguo` → `Training.TrainEpochBackpropTokenized` / `ValidateTokenized`；
  - 三国 GPU：`Program.TrainSanguoGpu` → `SanguoGpuTrainer.TrainOnGpu`（TorchSharp + CUDA），训练完导出为 C# 模型权重。

- **推理入口**
  - 数字任务：`dotnet run -- infer` 或直接调用 `InferenceExample.Run`；
  - 三国问答：`dotnet run -- ask [模型目录]`，内部使用 `SanguoInference.RunInteractive`。

### 四、建议阅读顺序（学习路径）

1. **Program.cs**  
   理解入口参数与几种模式的分支逻辑（train / infer / sanguo / generate-sanguo / ask 等）。
2. **Data/**  
   先看 `Vocabulary` 与 `DataGenerator`（数字任务），再看 `CharVocabulary` 与 `SanguoQaDataLoader`、`SanguoQaDataGenerator`（文本任务）。
3. **Core/MatrixHelper.cs**  
   熟悉矩阵运算、Softmax、LayerNorm、交叉熵及其 Backward 的实现。
4. **Model/**  
   按依赖顺序阅读：Embedding → 注意力与 FFN → Encoder/Decoder 层 → `TransformerModel`。
5. **Training/**  
   理解两种训练方式（SPSA 与手写 Backprop+SGD）、验证与评估指标。
6. **Inference/**  
   看数字任务推理示例 `InferenceExample`，再看三国问答推理 `SanguoInference`。
7. **docs/**  
   查看补充文档（架构总览、训练与推理流程、数据结构说明等），从更高层次回顾整个项目。
